容器相关操作

  • 启动容器 docker run

    | 参数 | 含义简介 |
    | ————————- | —————————————————— |
    | -t | 在新容器内指定一个伪终端或终端。 |
    | -i | 允许你对容器内的标准输入 (STDIN) 进行交互。 |
    | -d | 在后台启动一个容器，所有后台容器都有一个唯一的容器ID run the container in detached mode (in the background) |
    | -p 80:80 | map port 80 of the host to port 80 in the container |
    | docker/getting-started | 使用的镜像，可以使用镜像名称也可以使用hash码 |
    | -v /宿主机目录:/容器目录 | 将宿主机的目录挂载到启动的容器内 |
    | -u | 指定登陆容器后的用户名称 ( -u root 可以获取容器内root权限)|

  • 查看运行中的容器 docker ps

    | Title | 含义 |
    | ———— | ——————————————- |
    | CONTAINER ID | 容器 ID。 |
    | IMAGE | 使用的镜像。 |
    | COMMAND | 启动容器时运行的命令。 |
    | CREATED | 容器的创建时间。 |
    | STATUS | 容器状态。 |
    | PORTS | 容器的端口信息和使用的连接类型（tcp\udp）。 |
    | NAMES | 自动分配的容器名称。 |

  • 进入容器 docker attach/exec

    进入容器时可以使用容器的ID（CONTAINER ID）也可以使用容器名称（NAMES）
