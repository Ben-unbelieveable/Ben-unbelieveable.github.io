Neo4j-04.数据导入与对接工具

发表于 2026-06-10 | 分类于 图数据库
Neo4j 数据导入全路径：LOAD CSV、neo4j-admin bulk import、APOC、官方驱动、NetworkX/pandas 转换及 Kafka/Debezium 同步方案。

把外部数据写入 Neo4j 是项目落地的第一步。本文按数据规模与在线/离线需求，梳理常见存储格式、导入方式与对接工具——从小文件 CSV 到亿级边 bulk import，再到 Python 驱动与 CDC 流式同步。

段末注释：ETL（extract, transform, load，抽取、转换、加载）指将源系统数据清洗后载入目标库的流程；CDC（change data capture，变更数据捕获）指捕获源库增量变更并同步到下游。

前置：Neo4j-03.Cypher增删改查

十、导入 checklist

  1. 建模确认：节点唯一键、关系类型、属性类型
  2. 先建约束/索引，再导数据
  3. 先导节点，再导边
  4. 百万级以上评估 bulk import 或 IN TRANSACTIONS
  5. 导入后 MATCH (n) RETURN count(n) 与源表对账
  6. 监控堆内存与事务日志空间
